# Interview Answers

## Q1. 실제로 로봇을 만들었나요?

A. 아닙니다. 현재 구현은 근력 보조 로봇이나 외골격 제어 모델이 아니라, 하네스 결합형 웨어러블 안전 시스템의 fall-risk 감지 모델입니다. 포트폴리오 분석 단계에서는 공개 IMU fall-detection 데이터와 과거 풍속 데이터를 직접 수집해, 합성 데이터가 아니라 실제 공개 데이터 기반으로 위험 감지 구조를 정리했습니다.

## Q2. 실제 AI 모델을 학습했나요?

A. 네. 공개 IMU 센서 기록을 window-level feature로 변환한 뒤 `normal_activity`와 `fall_risk`를 분류하는 RandomForest 모델을 학습했습니다. 단, 이 모델은 Peter Anchor 하드웨어나 실제 고층 외벽 청소 현장 성능을 검증한 것은 아닙니다.

## Q3. 어떤 실제 데이터를 사용했나요?

A. IMU 데이터는 `jasonkau/fall-detection-dataset-IMU`의 공개 엑셀 파일을 수집해 사용했습니다. 풍속 데이터는 Open-Meteo Historical Weather API에서 서울 지역의 시간별 풍속과 돌풍 데이터를 가져왔습니다.

## Q4. 왜 로프 장력이나 하네스 압력은 모델에 없나요?

A. 해당 값들은 Peter Anchor 같은 특정 하드웨어나 실제 작업 현장에서 직접 계측해야 하는 데이터입니다. 공개된 신뢰 가능한 로프 장력·하네스 압력 시계열 데이터를 찾기 어려워, 임의로 만들지 않고 향후 수집 필요 항목으로 분리했습니다.

## Q5. 본인의 기여는 무엇인가요?

A. 문제 정의, 위험 요인 분류, 실제 공개 데이터 소스 조사, 데이터 수집 스크립트 구현, IMU feature 변환, fall-risk 분류 모델 학습, 검증 한계 정리를 수행했습니다.

## Q6. 이 프로젝트의 한계는 무엇인가요?

A. 공개 IMU 데이터는 실제 센서 기록이지만 고층 외벽 청소 현장 데이터는 아닙니다. 풍속 역시 지역 기상 데이터이므로 건물 외벽 근접 풍속을 완전히 대체할 수 없습니다. 산업 적용을 위해서는 실제 하네스/로프 센서 데이터 수집, 안전 인증, 현장 파일럿 테스트가 필요합니다.

## Q7. 왜 웨어러블 로봇이라고 보기 어렵나요?

A. 현재 모델은 위험 상태를 감지할 뿐, 근력 보조나 관절 보조 토크를 생성하지 않습니다. 웨어러블 로봇이라고 하려면 EMG, 관절각, 관절토크, actuator torque 데이터를 기반으로 사용자의 움직임 의도나 필요한 보조 토크를 예측하고, 실제 구동부가 힘을 보조하는 구조가 필요합니다.

## Q8. 웨어러블 로봇으로 확장하려면 어떤 모델이 필요한가요?

A. fall-risk 감지 모델과 별도로 `assistive torque prediction` 또는 `muscle-assist control` 모델이 필요합니다. 예를 들어 EMG와 관절각/각속도 데이터를 입력으로 받아 무릎, 고관절, 발목 또는 상지 관절의 필요한 보조 토크를 예측하는 회귀 모델을 만들 수 있습니다.

## Q9. 원래 기획의 가방형 그물망과 현재 AI 모델은 어떻게 연결되나요?

A. 현재 AI 모델은 `fall_risk`를 감지하는 역할을 합니다. 가방형 그물망은 그 감지 결과를 받아 작동하는 emergency protection layer로 연결할 수 있습니다. 다만 모델이 바로 전개를 결정하는 것이 아니라, fall-risk 확률, 그물망 준비 상태, 하네스 연결 여부, 작업 높이, 수동 override를 함께 보는 safety supervisor가 `warning`, `arm_net`, `deploy_net` 같은 행동을 결정하는 구조가 적절합니다.

## Q10. 그물망이 있으면 웨어러블 로봇이라고 할 수 있나요?

A. 그물망이 자동 전개되는 구동부를 가진다면 능동 안전장치에 가까워질 수는 있지만, 근력 보조 로봇이라고 보기는 어렵습니다. 그물망은 관절 토크를 보조하는 장치가 아니라 추락 시 충격을 완화하는 보호 장치입니다. 그래서 현재는 wearable safety system으로 설명하고, 로봇 확장은 assistive torque control을 추가하는 방향으로 분리했습니다.

## Q11. accuracy보다 어떤 지표를 중요하게 봤나요?

A. 안전 관련 문제에서는 전체 accuracy보다 위험 클래스를 놓치지 않는 recall과 f1-score가 더 중요합니다. 그래서 fall-risk recall과 macro F1을 함께 확인했습니다.

## Q12. 이 프로젝트를 포트폴리오에서 어떻게 설명할 건가요?

A. 하네스 결합형 AI 안전 웨어러블 시스템 아이디어를 실제 공개 IMU 데이터와 풍속 데이터 기반 분석으로 확장한 프로젝트라고 설명하겠습니다. 현재 구현은 fall-risk 감지와 가방형 그물망 response policy까지이며, 로봇 또는 외골격으로 확장하려면 근력 보조 제어 데이터와 actuator 검증이 추가로 필요하다는 점을 명확히 밝히겠습니다.
